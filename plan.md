# t-cli 终端翻译工具实现方案

## 1. 项目基础信息
*   **项目/文件夹名**: `t-cli`
*   **全局命令**: `t-cli`
*   **核心包依赖**:
    *   原生 Node.js `fetch` (Node >= 18) 进行无状态网络请求，无需 axios。
    *   `chalk`（用于终端输出的高亮和美化）。
    *   `dotenv`（可选，支持从项目根目录的 `.env` 文件加载环境变量）。

## 2. 目录与文件结构
```text
t-cli/
├── bin/
│   └── t-cli.js         # CLI 入口文件，定义 shebang
├── src/
│   ├── api.js           # 封装 DeepSeek 接口，仅传入当前 system 提示词和 user 文本
│   ├── prompts.js       # 管理中英互译、语法检查两套 System Prompts
│   └── repl.js          # 核心对话循环，处理命令行输入输出
├── .env.example         # 环境变量配置示例
├── package.json         # 定义 "bin": { "t-cli": "./bin/t-cli.js" }
├── README.md            # 项目命令说明、安装及使用指南
└── plan.md              # 方案落地文档（即本文档）
```

## 3. 核心运行逻辑
1.  **启动阶段**: 用户在终端输入 `t-cli`。程序读取环境变量 `DEEPSEEK_API_KEY`，若未配置则以红色高亮提示报错并退出，给出配置命令示例。
2.  **交互阶段**: 使用原生 `readline` 模块开启命令行监听，显示提示符（如 `t-cli > `）。
3.  **模式路由**:
    *   **语法检查模式**: 拦截以 `/check ` 开头的输入。提取后面的英文句子，结合预设的“英语老师/润色” Prompt 发送给 DeepSeek API，不携带历史上下文。
    *   **自动互译模式**: 对于其他普通输入，结合预设的“专业翻译” Prompt 发送，DeepSeek 会自动识别语种并返回地道的美式英语或流畅的中文。
    *   **退出命令**: 拦截 `exit`、`quit` 关键词或 `Ctrl+C` 事件以优雅退出。
4.  **展示阶段**: 接收到 API 响应后，使用 `chalk` 将翻译结果或检查建议高亮输出在终端，并重新展示输入提示符，等待下一次输入。

## 4. 开发与安装说明 (记录于 README)
*   **本地开发测试**: 在项目目录下执行 `npm link` 即可在全局使用 `t-cli`。
*   **环境变量配置**:
    *   可以新建 `.env` 文件。
    *   或者配置全局环境变量：`export DEEPSEEK_API_KEY="your_api_key"`。
